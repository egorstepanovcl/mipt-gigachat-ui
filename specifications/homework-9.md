## Задание

**Ожидаемый результат:** приложение работает в интернете по публичной ссылке.

> ⚠️ Необязательно реализовывать все пункты оптимизации — главное развернуть рабочее приложение с подключённым GigaChat API и оформленным README.

### 1. Аудит бандла

Проанализируйте размер и состав итоговой сборки:

- подключите `vite-bundle-visualizer` (`npx vite-bundle-visualizer`) или `source-map-explorer`;
- найдите самые тяжёлые зависимости в бандле;
- убедитесь, что `react-markdown`, `highlight.js` / `Prism.js` не попадают в основной чанк;
- добавьте скриншот визуализатора бандла в папку `docs/` или в `README`.

### 2. Оптимизация ререндеров

Примените мемоизацию там, где это действительно необходимо:

- оберните `ChatItem` в `React.memo` — список чатов не перерисовывается при изменении другого чата;
- используйте `useMemo` для отфильтрованного списка чатов в поиске;
- используйте `useCallback` для обработчиков `onSend`, `onDelete`, `onRename`, передаваемых в дочерние компоненты;
- проверьте результат через вкладку **Profiler** в React DevTools.

### 3. Code Splitting с React.lazy + Suspense

Разбейте приложение на асинхронные чанки:

- загружайте `Sidebar` лениво через `React.lazy` + `Suspense` с fallback-заглушкой;
- загружайте `SettingsPanel` лениво — она открывается редко и не нужна при старте;
- убедитесь, что отдельные роуты (`/`, `/chat/:id`) загружаются отдельными чанками.

### 4. Error Boundaries

Изолируйте ошибки, чтобы сбой в одном компоненте не останавливал всё приложение:

- реализуйте компонент `ErrorBoundary` (классовый компонент с `componentDidCatch`);
- оберните область сообщений `MessageList` в `ErrorBoundary` — ошибка рендера не ломает сайдбар;
- оберните API-вызовы в блоки `try/catch` с выводом `ErrorMessage` под полем ввода;
- добавьте кнопку «Повторить» в UI-ошибки.

### 5. Деплой приложения

Опубликуйте приложение в интернете:

- разверните приложение на **Vercel**, **Netlify** или **GitHub Pages** — на выбор;
- убедитесь, что переменные окружения (токен GigaChat API) добавлены в настройки хостинга через env-переменные, **а не размещены в коде**;
- проверьте, что роуты React Router работают корректно на хостинге (настройте `redirects` для Netlify или `vercel.json` для Vercel);
- проверьте работу приложения в режиме инкогнито.

### 6. Оформление README

Файл `README.md` должен содержать:

- **Демо** — публичная ссылка на развёрнутое приложение + скриншоты или видеозапись работы.
- **Стек** — список технологий с версиями (React, TypeScript, React Router, стейт-менеджер, стилизация).
- **Запуск локально** — пошаговая инструкция: `git clone`, `npm install`, заполнение `.env`, `npm run dev`.
- **Переменные окружения** — таблица с описанием всех `.env`-переменных (без реальных значений).
