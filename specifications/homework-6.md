**Тема:** Управление состоянием приложения: Context, useReducer, внешний state-менеджмент

**Время выполнения:** 2 академических часа

Вынести состояние приложения в глобальный стор, реализовать работу с несколькими чатами и добавить навигацию между ними. Подключить реальный GigaChat API вместо заглушек и обеспечить корректную работу контекста диалога и рендеринга ответов с markdown и подсветкой кода.

## Задание

> ⚠️ Если не получается streaming/SSE, допустимо использовать обычный REST-ответ, но интеграция с GigaChat API обязательна для выполнения итогового задания. Необязательно реализовывать все пункты — главное соблюдать требования итогового задания.

### 1. Глобальное состояние чатов

Перенесите состояние из локальных хуков в глобальный стор:

- Используйте **Context API + useReducer** или один из стейт-менеджеров: **Zustand / Redux Toolkit** на выбор.
- В сторе храните: список чатов, активный чат, сообщения текущего чата, состояние загрузки и ошибок.
- Определите TypeScript-типы сущностей: `Chat`, `Message`, `ChatState`, `ChatAction`.

### 2. Боковая панель и управление чатами

Доработайте `Sidebar` и связанные компоненты:

- Отображение списка всех чатов пользователя.
- Создание нового чата по кнопке «Новый чат».
- Переключение между чатами без потери истории сообщений.
- Редактирование названия чата.
- Удаление чата с подтверждением (диалог или модальное окно).
- Поиск по чатам по названию и содержимому последнего сообщения.

### 3. Персистентность с `localStorage`

Реализуйте сохранение истории чатов между перезагрузками:

- При изменении стейта данные сохраняются в `localStorage`.
- При инициализации приложения состояние загружается из `localStorage`, если оно есть.
- Обрабатывается кейс «битых» данных (`try/catch` при `JSON.parse`).

### 4. Маршрутизация между чатами

Добавьте маршрутизацию с помощью **React Router**:

- Маршрут `/` — список чатов, может отображать последний открытый чат или пустое состояние.
- Маршрут `/chat/:id` — открытый чат с конкретным идентификатором.
- При переходе по URL напрямую (вкладка/refresh) чат восстанавливается по `id` из адресной строки.
- При удалении активного чата пользователь перенаправляется на `/`.

### 5. Автогенерация названия чата

- При создании чата название генерируется по первому сообщению, обрезанному до 30–40 символов.
- Если первое сообщение пустое или слишком короткое — используется название по умолчанию (`«Новый чат»`, `«Диалог N»`).

### 6. Интеграция с GigaChat API ⚠️ Обязательно

Замените заглушки на реальные запросы:

- Вместо моков — запрос к GigaChat API.
- Передача полного контекста диалога: массив `messages` с ролями `system` / `user` / `assistant`.
- Реализация **streaming/SSE-режима** (`stream: true`) для постепенного отображения ответа; если не получилось — обычный REST-запрос с полным ответом.
- Обновление сообщения ассистента по мере прихода чанков (streaming) или целиком (REST).

### Markdown и подсветка кода

- Подключите `react-markdown` или аналог для рендеринга ответов ассистента.
- Добавьте подсветку кода через `highlight.js` или `Prism.js` для блоков кода в markdown *(по желанию)*.
