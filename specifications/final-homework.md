## Цель задания

Проверить знания и навыки, полученные в процессе освоения дисциплины «Основы frontend-разработки».

## Описание задания

Создайте приложение, повторяющее современные паттерны интерфейса ChatGPT, но работающее на базе интеграции с публичным API GigaChat. Приложение должно предоставлять интерфейс для диалога с нейросетью, сохранять историю чатов, управлять настройками и обеспечивать удобный пользовательский опыт.

## Инструменты

**Основные инструменты:** React, JavaScript, API GigaChat, GitHub или его аналог.

> Подробнее — в разделе [Технические требования](#технические-требования).

## Функциональные требования

### Интерфейс чата

- Реализовать главный экран с областью отображения сообщений и полем ввода запроса.
- Отображать сообщения пользователя и ассистента в хронологическом порядке с визуальным разделением: разный дизайн для `user` и `assistant`.
- Поддерживать Markdown-форматирование в ответах GigaChat (заголовки, списки, код, ссылки).
- Реализовать индикацию загрузки (typing indicator) во время обработки запроса нейросетью.
- Обеспечить автоматическую прокрутку к последнему сообщению при получении ответа.
- Предусмотреть возможность копирования ответа ассистента в буфер обмена.
- Реализовать функцию «Остановить генерацию» для прерывания длинных ответов.

### Управление чатами

- Создать боковую панель (sidebar) со списком всех чатов пользователя.
- Реализовать создание нового чата с автоматической генерацией названия на основе первого сообщения.
- Обеспечить возможность переключения между чатами без потери данных.
- Добавить функции редактирования названия чата, удаления чата с подтверждением.
- Реализовать поиск в истории чатов по названию и содержимому.
- Сохранять историю сообщений каждого чата в `localStorage` или `IndexedDB`.

### Работа с GigaChat API

- Реализовать отправку запросов к эндпоинту `POST /api/v1/chat/completions` с корректными заголовками и телом запроса.
- Передавать контекст диалога (массив `messages` с ролями `system`/`user`/`assistant`) для поддержания связности беседы.
- Реализовать streaming-режим (`stream: true`) для постепенного отображения ответа токен за токеном.
- Обрабатывать Server-Sent Events (SSE) при использовании streaming.
- Поддерживать настройку параметров запроса: `temperature`, `top_p`, `max_tokens`, `repetition_penalty`.
- Реализовать обработку изображений через GigaChat API (multimodal capabilities).

## Технические требования

### Стек технологий

| Категория | Инструменты |
|---|---|
| Фреймворк | React 18+ с TypeScript |
| State management | Context API + `useReducer`, Zustand, Redux Toolkit или Jotai |
| Стили | CSS Modules, Styled Components, Tailwind CSS или SCSS |
| HTTP-клиент | Fetch API или Axios |
| Markdown Rendering | `react-markdown` или `marked.js` |
| Подсветка синтаксиса | `Prism.js` или `highlight.js` |

### Архитектура

- Использовать Feature-Sliced Design или модульную архитектуру с разделением на слои.
- Выделить отдельный API-слой для работы с GigaChat (паттерн «адаптер»).
- Реализовать кастомные хуки для работы с чатами, сообщениями и потоковым режимом.
- Использовать паттерн «компоновщик» для переиспользуемых UI-компонентов.
- Реализовать Error Boundaries для изоляции ошибок в компонентах.

### Интеграция с GigaChat API

- **Базовый URL:** [gigachat.devices.sberbank.ru/api/v1/](https://gigachat.devices.sberbank.ru/api/v1/)
- **OAuth-эндпоинт:** `POST` [ngw.devices.sberbank.ru:9443/api/v2/oauth](https://ngw.devices.sberbank.ru:9443/api/v2/oauth)
- **Эндпоинт чата:** `POST /api/v1/chat/completions`
- **Эндпоинт модели:** `GET /api/v1/models`

**Обязательные заголовки:**

```http
Authorization: Bearer <token>
Content-Type: application/json
Accept: application/json
```

**Формат тела запроса к `/chat/completions`:**

```json
{
  "model": "GigaChat",
  "messages": [
    {"role": "system", "content": "Ты полезный ассистент"},
    {"role": "user", "content": "Привет!"}
  ],
  "temperature": 1.0,
  "top_p": 0.9,
  "max_tokens": 2048,
  "stream": true
}
```

## Этапы реализации

### Этап 1 — Базовый чат

- Создание компонентов: `ChatWindow`, `Message`, `InputField`.
- API-слой: функция `sendMessage` к `/chat/completions`.
- State management для массива `messages`.
- Отображение сообщений пользователя и ассистента.
- Базовая обработка ошибок.

### Этап 2 — Streaming и UX

- Реализация потокового режима с `EventSource`/`fetch`.
- Парсинг SSE и постепенное отображение ответа.
- Индикатор ввода (typing indicator), автоскролл.
- Markdown-рендеринг с `react-markdown`.
- Подсветка кода в блоках.

### Этап 3 — Управление чатами

- Боковая панель со списком чатов.
- CRUD-операции для чатов.
- Персистентность в `localStorage` и `IndexedDB`.
- Переключение между чатами.
- Поиск по чатам.
