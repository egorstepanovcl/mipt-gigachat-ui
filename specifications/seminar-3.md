# Задача

Реализуйте компонент input для чат‑интерфейса на React, поддерживающий отправку сообщений и дополнительные функции, удобные для И‑чата.

Создайте компонент `ChatInput` на React, похожий по поведению на поле ввода в ИИ-чате. Компонент будет использоваться в итоговом роекте — чат‑приложении: сверху история сообщений, снизу `ChatInput`.

## Обязательные требования

### 1. Поле ввода на основе `<textarea>`

- Многострочный ввод
- Автоизменение высоты до заданного максимума, например 30–40% высоты окна

### 2. Управление клавиатурой

- `Enter` — отправка сообщения
- `Shift + Enter` — перенос строки без отправки сообщения

### 3. Кнопка отправки

- Кнопка неактивна, если поле пустое или идёт отправка
- Иконка «Отправить» — можно реализовать простую стрелку или текст `Send`

### 4. Состояния загрузки

- Во время ответа ассистента кнопка блокируется: текст вводить можно, но отправка запрещена, чтобы имитировать поведение ИИ-чата

## Дополнительно

### 1. Поддержка горячих клавиш

- `Ctrl + K` / `Cmd + K` — очистить поле ввода
- `Esc` — снять фокус и очистить поле

### 2. Отображение лимита длины сообщения

- Максимальное количество символов, индикатор оставшихся символов

### 3. Индикация «пользователь печатает»

- Колбэк `onTyping` вызывается при вводе, чтобы родитель мог показывать статус `typing…`

### 4. Минимальная поддержка drag & drop файлов

- Отлов события `onDrop` и передача списка файлов наружу через пропсы `onFilesDrop`

## API компонента

```ts
type ChatInputProps = {
  value: string;
  onChange: (value: string) => void;
  onSend: (message: string) => void;   // Вызывается при отправке
  disabled?: boolean;                  // Блокировка при загрузке ответа
  placeholder?: string;
  maxRows?: number;                    // Для авторесайза textarea
  maxLength?: number;                  // Лимит символов
  onTyping?: () => void;               // Опционально: уведомить, что пользователь печатает
  onFilesDrop?: (files: FileList) => void; // Опционально: drag & drop вложений
};
```

## Условия

- Компонент **контролируемый**: значение приходит из `props.value`, изменения — через `onChange`
- `onSend` не должен вызываться, если строка пустая или состоит только из пробелов
- После успешной отправки поле очищается и высота `textarea` сбрасывается
